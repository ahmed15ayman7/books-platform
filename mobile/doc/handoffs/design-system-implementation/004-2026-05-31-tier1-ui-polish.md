## Notes

- **`BookCardShimmer` with its own `Shimmer.fromColors`** — each instance runs an independent animation cycle. Multiple cards shimmering slightly out of phase is acceptable UX; the alternative (one parent `Shimmer` wrapping all children) would require splitting shimmer layout from animation and complicating the widget hierarchy.
- **`AnimatedSwitcher` key pattern** — every branch of every BlocBuilder switch now carries a `ValueKey`. OR-pattern states (e.g. `PublisherDetailLoading() || PublisherDetailInitial()`) share `ValueKey('loading')` — correct because they map to the same UI. Detail screens keep `AppLoadingIndicator` for the loading state; only list screens get shimmer skeletons.
- **`_DetailBody` in `book_detail_screen.dart` does not expose `super.key`** — use `KeyedSubtree(key: ..., child: _DetailBody(...))` pattern if you need to key it. This is documented here to avoid re-hitting the same compile error.
- **`RefreshIndicator` on `CustomScrollView` / `GridView`** — all RefreshIndicator parents have `physics: const AlwaysScrollableScrollPhysics()` set explicitly. Without this, pull gesture fails when list content is shorter than the viewport.
- **`ArticlesListCubit.refresh()`** uses a Dart 3 switch expression to read `activeChannel` from the current state — no new field was added to the cubit. If the state is not `ArticlesListSuccess` when refresh is called, it defaults to `'harvest'` (the `load()` default).
- **`SearchScreen` private widgets** (`_RecentChips`, `_ResultsList`, `_NoResults`) do not expose `super.key`. They are wrapped with `KeyedSubtree(key: ValueKey(...), child: ...)` for AnimatedSwitcher. Any new private widget in this file should follow the same pattern.
- **All 14 screens remain functional** — `flutter analyze` → 0 issues confirmed at end of session.
